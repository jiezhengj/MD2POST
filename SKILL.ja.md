# MD2POST — Skill リファレンス（AI Agent 専用）

> 本ドキュメントは AI Agent が読むために設計されています。Codex、Claude Code、Open Code、Gemini CLI、Antigravity、Trae などあらゆる主流モデルフレームワークが、環境準備・実行・統合を自律的に完了できることを目標としています。

---

## I. スキルの位置づけ

MD2POST は**「筋肉型」確定性実行ツール**であり、職責の範囲は明確です：

- **入力**：ローカル `.md` ファイルパス + オプションのテーマ・言語パラメータ
- **出力**：指定した出力ディレクトリに保存される高解像度 PNG 長編画像（1枚以上）

Agent の責任は入力ファイルを準備してこのツールを呼び出すことです。MD2POST の責任は確定的な方法でテキストから画像への完全変換を行うことであり、外部 API やネットワーク権限は不要です。

---

## II. システム前提条件

このスキルを呼び出す前に、ホストマシンが以下の要件を満たしていることを確認してください：

| 依存関係 | バージョン要件 | 備考 |
|---------|-------------|------|
| **Node.js** | ≥ 18 LTS | ESM (`"type": "module"`) のサポートが必要 |
| **npm** | ≥ 9 | 依存関係のインストール用 |
| **Chromium または Chrome** | Playwright が自動管理 | ヘッドレスブラウザスクリーンショット用 |

---

## III. インストール手順（初回使用時）

以下のコマンドを順番に実行してください。各コマンドが正常に完了したことを確認してから次へ進んでください。

### ステップ 1：Node 依存関係のインストール

```bash
npm install
```

### ステップ 2：Playwright のヘッドレスブラウザコアのインストール

このステップでは Chromium をダウンロードします。約 200MB のディスク容量とネットワーク接続が必要です。

```bash
npx playwright install chromium
```

### ステップ 3：インストールの確認

```bash
npx tsx src/index.ts --help
```

パラメータの説明が出力されれば，环境の準備完了です。

### ステップ 4：Vendor モジュールのビルド (beautiful-mermaid fork)

本プロジェクトには Issue #83 修正（TD/TB レイアウト反転バグ）を含む `beautiful-mermaid` ローカルフォークが含まれています。ビルドが必要です：

```bash
cd vendor/beautiful-mermaid
npm install
npm run build
cd ../..
```

## IV. CLI インターフェース

### 基本フォーマット

```bash
npx tsx src/index.ts -i <入力ファイル> [オプション]
```

### 完全パラメータ表

| パラメータ | 短縮名 | 必須 | デフォルト | 有効値 | 説明 |
|----------|-------|------|----------|--------|------|
| `--input` | `-i` | はい | — | 任意の `.md` ファイルパス | 入力 Markdown ファイル（相対パスまたは絶対パス） |
| `--output` | `-o` | いいえ | `./out` | 任意のディレクトリパス | 出力ディレクトリ（自動作成） |
| `--theme` | `-t` | いいえ | `humanities` | `tech` / `humanities` / `emotion` | ビジュアルスタイルテーマ |
| `--lang` | `-l` | いいえ | `zh` | `zh` / `ja` / `en` | ベース言語（フォントサイズと組版密度に影響） |

### 使用例

```bash
# 最小限の呼び出し（デフォルトテーマと言語を使用）
npx tsx src/index.ts -i ./my_article.md

# 青色テクノロジーテーマ + 英語を指定
npx tsx src/index.ts -i ./report.md -t tech -l en -o ./output

# エモーションテーマ + 日本語を指定
npx tsx src/index.ts -i ./post.md -t emotion -l ja
```

## V. 3種類のテーマの説明

| テーマ ID | 名称 | スタイル説明 | 適したコンテンツ |
|---------|------|------------|--------------|
| `tech` | テクノロジー | クールホワイト背景 + 青紫ハイライト | 技術共有、製品分析 |
| `humanities` | 人文 | クリーミーウォームホワイト + 赭石アクセント | 書評、エッセイ、人文コンテンツ |
| `emotion` | エモーション | ダーク背景 + ゴールドハイライト | 感想、詩、感情的なコンテンツ |

## VI. サポートする Markdown 記法

以下の記法はこのエンジンで完全サポートされ、視覚的に精調されています：

**標準記法**
- 順序付き / 順序なしリスト、ネストされたインデント
- 太字、斜体、取り消し線
- インラインコード、コードブロック（シンタックスハイライトコンテナ付き）
- 引用ブロック
- テーブル
- 画像（**ローカル相対パスのみ** — HTTP リモート画像は禁止）
- 水平線 `---`

**拡張記法**（公式プラグインによる）
- `==ハイライトテキスト==` — ゴールド丸角ハイライトマーク
- `^上付き^` / `~下付き~` — 精密ベースライン整列の上付き・下付き文字
- `- [ ] 未完了` / `- [x] 完了` — 丸角チェックボックスタスクリスト

**Mermaid 図表**
- コードブロックで `` ```mermaid `` を使用します。
- エンジンは AST フェーズで `beautiful-mermaid` を同期的に呼び出して高忠実度の SVG ベクター画像をコンパイルします。レンダリング遅延なし、品質劣化なし。
- サポート: `graph TD/LR`、`flowchart`、`stateDiagram-v2`、`sequenceDiagram`、`classDiagram`、`erDiagram` などの主流タイプ。
- **注意**：Mermaid の記法ヘッダーにセミコロンを追加しないでください（例：`graph TD;` は無効 — `graph TD` を使用してください）。

## VII. 出力の説明

正常実行後、出力ディレクトリには以下のファイルが含まれます：

| ファイル名 | 説明 |
|----------|------|
| `debug_phase1.html` | プレビュー用 HTML ファイル（ブラウザで開いて組版確認可能） |
| `post.png` | コンテンツ高さが 8000px 以下の場合の単一出力画像 |
| `post_1of3.png` など | コンテンツが 8000px を超える場合、セマンティックセーフポイントで分割されたセグメント画像 |

## VIII. 容量制限とエラー処理

| 状況 | トリガー条件 | エンジンの動作 |
|-----|------------|-------------|
| 正常 | 推定高さ < 24000px | 通常レンダリング |
| 警告 | 推定高さ 24000〜32000px | `[WARNING]` を出力、マルチ画像スライスモードを起動、実行継続 |
| 致命的エラー | 推定高さ > 32000px | `[CAPACITY_EXCEEDED]` を出力して終了、削減すべき文字数を提示 |
| リモート画像 | Markdown に `http://` または `https://` の画像が含まれる場合 | 即座にエラーを出力して終了 — ネットワークリクエストは禁止 |

## IX. Agent 統合ガイドライン

1. **コンテンツの事前生成**：Agent はまずユーザーコンテンツを標準 Markdown に整理して一時ファイルに書き込み、その後このツールを呼び出すことができます。
2. **パラメータの決定**：テーマ（`--theme`）と言語（`--lang`）が Agent が動的に渡せる唯一のパラメータです。すべての組版規則（フォントサイズ・行間・間隔）はエンジン内部に固定されており、Agent は上書きできません。
3. **画像参照**：コンテンツに挿入画像が含まれる場合は、画像ファイルがすでに相対パスでローカルに書き込まれ、Markdown で正しく参照されていることを確認してください（例：`![説明](./images/chart.png)`）。
4. **エラー回復**：`[CAPACITY_EXCEEDED]` が発生した場合、Agent は元のテキストを自動的にセグメントに分割し、このツールを複数回呼び出して複数の画像グループを生成する必要があります。
